Major Assignment FAQ

Please check this FAQ page regularly as this page will be constantly updated in response to common questions.

Data cleaning stage

How many data cleaning errors do we need to find?

There are approximately 5 data errors in the MonEquip database.

How do we count the errors to be different?

Firstly, it will be counted as 1 error as you only need to use 1 SQL statement to find that error.
Secondly, errors in different tables (e.g. duplicate values in two different tables), then it would be considered as two different errors. One for each table.

What do we do with the data errors?

If the error interferes with the calculation of your fact measure and/or is not consistent with the values in the database, then you can remove the record(s). However, if the errors related to the miscalculation in the formula, you should update the records using the correct formula instead of deleting the records.

Can we change the Operational Database?

You must not change the Operational Database at all, except for data cleaning purpose.

Can we add more tables in the Operational Database?

You must not add any new tables/attributes based on your assumptions.

Can we add new PKs to the original operating database?

You must not add new PK to the Operational Database.

Will adding constraints to the operational database, be considered as error correction?

You don't need to create constraints for the tables since the operational database has been created by the management for you. In addition, you can only "clean" the data through the data cleaning process (e.g. removing duplicates). See Session 2 Moodle resources for more details.

Can the data cleaning part be done using different tools such as Python etc.?

For this unit's assignment, please use SQL to clean the data.

How do we import the operational database to our account?

You can use this approach :

create table <tablename> as select * from <username.tablename>;

Designing the data warehouse by drawing star/snowflake schema

What drawing tools should we use to draw the Star Schema?

You can use Lucidchart or draw.io.

Are both the star schemas expected to address all the questions?

Yes both star schemas should be able to answer all those questions.

In the specification, it states that "For each attribute, you may apply your own design decisions on specifying a range or a group, but make sure to specify them in your submission." So are we allowed to add more dimensions for this case?

Yes, you can add more dimensions apart from the dimension attributes listed in the specification. However, please ensure that you specify them in your report.

For the dimensions mentioned in the assignment specification, do we have to include all of them in one star schema? Can we use some of them in Star schema version1, and the missing dimensions will be added in version2?

You are expected to include all dimensions in both versions of the star schemas, otherwise, your level 0 and high level star schema are not comparable.

Can I have a fact table without any aggregation in a level 2 star schema?

A level 2 star schema can have different fact tables at different levels of aggregation, even level 0. Therefore, it is possible to have some fact tables at level 0. However, if there is no aggregation at all fact and dimension tables, that star schema should be at level 0.
